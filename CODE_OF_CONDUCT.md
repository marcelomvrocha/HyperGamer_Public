# Code of Conduct

## Our Standard

Contributors are expected to keep discussion respectful, direct, and constructive.

Examples of behavior that help this project:

- assuming good intent while reviewing technical work
- giving actionable feedback instead of vague dismissal
- focusing criticism on code and decisions, not people
- respecting different experience levels and backgrounds

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- discriminatory language or exclusionary behavior
- deliberate intimidation, dogpiling, or bad-faith engagement
- publishing private information without consent

## Enforcement

Project maintainers may remove comments, reject contributions, or block participation when behavior undermines a safe and productive collaboration environment.

## Reporting

If you need to report a conduct issue, use the contact path described in [SECURITY.md](./SECURITY.md) and clearly mark the message as a conduct concern.
